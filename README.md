# MDGreet - Material Design Greeter

A greetd greeter with full Material Design theming support.

## Features

- **Dynamic Color Scheme**: Full Material Design 3 color scheme support with light/dark modes
- **Theme Switching**: Switch between multiple themes at runtime
- **Custom Themes**: Support for custom themes generated by Material Theme Builder
- **Configurable**: TOML-based configuration for customization

## Theme System

The theme system uses original Material Theme Builder JSON files without modification.

### Available Built-in Themes

- **slint** - Default Slint theme (blue)
- **purple** - Material's signature purple
- **green** - Nature-inspired green palette
- **red** - Bold and vibrant red theme

### Configuration

The configuration file uses TOML format. By default, it is located at `/etc/greetd/mdgreet.toml`.
You can use a custom config file with the `--config` argument:

```bash
mdgreet --config /path/to/custom/config.toml
```

A sample configuration is provided in `mdgreet.sample.toml`.

```toml
[theme]
name = "purple"    # or "slint", "green", "red", "custom"
mode = "dark"      # or "light"
```

### Custom Themes

To use a custom theme:

1. Generate a theme file using [Material Theme Builder](https://material-foundation.github.io/material-theme-builder/)
2. Save the exported JSON as `material-theme.json`
3. Set `name = "custom"` in your config.toml
4. Place the file in `MDGREET_CONFIG_DIR` (or current directory)

```bash
# Run with custom theme
MDGREET_CONFIG_DIR=/path/to/config cargo run
```

### Environment Variables

**Runtime:**
- `MDGREET_CONFIG` - Override config path (fallback if `--config` not provided)
- `MDGREET_CONFIG_DIR` - Directory for custom themes (used when `name = "custom"`)

**Compile-time (set during build):**
- `GREETD_CONFIG_DIR` - Override default config directory (default: `/etc/greetd`)
- `MDGREET_DEFAULT_THEME` - Override default theme (default: `purple`)
- `MDGREET_DEFAULT_MODE` - Override default color mode (default: `dark`)

```bash
# Build with custom defaults
GREETD_CONFIG_DIR=/etc/custom MDGREET_DEFAULT_THEME=green cargo build
```

### Architecture

```
src/
├── config.rs    # TOML configuration parsing
├── theme.rs     # Theme loading and application logic
└── main.rs      # Application entry point

ui/
├── main.slint   # UI definition
└── themes/      # Built-in theme JSON files (from Material Theme Builder)
```

## Building

```bash
# Using Nix (recommended)
nix develop
cargo build

# Or directly
cargo build
```

## Running

```bash
# Run with default config (/etc/greetd/mdgreet.toml)
cargo run

# Run with custom config
cargo run -- --config /path/to/config.toml

# Run with custom theme
MDGREET_CONFIG_DIR=/path/to/theme cargo run
```

## Technology Stack

- **Rust** - Core application logic
- **Slint** - UI framework with Material Design components
- **Material Design** - UI library from Slint's examples
- **TOML** - Configuration files
- **JSON** - Theme definitions (Material Theme Builder format)
